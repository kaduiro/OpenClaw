# ogawa-kogyo-hp 事前調査ノート

## プロジェクトの目的
小川工業株式会社のコーポレートサイト改善案件。特に `/works/` ページの改善が中心。現在の「強み」を訴求する実装から、打ち合わせで固まった「歴史」として会社の継承や厚みを伝える構成への変更が主な目的。

## 現在の実装構造
- メインリポジトリ: (ホストOS上の `C:/Users/akkun/kaduiro/ogawa-kogyo-hp` が、サンドボックス環境で `/repo` としてマウントされている想定)
- 主たる実装対象テーマ: `theme/ogawa-corp` (リポジトリルートからの相対パス)
- 関連テーマ（参照対象）: `theme/home5`, `theme/header-footer` (リポジトリルートからの相対パス)
- WordPress 側の実体テーマ: `wp-content/themes/home` (リポジトリ内のテーマ構成と WordPress 実体に差分がある可能性あり)
- 開発環境: Docker Compose, WordPress, Simply Static

## 主対象テーマと関連テーマの関係
主対象テーマは `theme/ogawa-corp` である。`theme/home5` と `theme/header-footer` は、過去の実装や関連するUIの参照元として存在する。これらのテーマから演出やUIを流用する可能性はあるが、共有コンポーネントやグローバルスタイルへの不用意な波及は避けるべきである。

## README 記述と実装実態の差分
リポジトリ直下の `README.md` には `theme/ogawa-renew` の記述があるが、現在の主対象テーマは `theme/ogawa-corp` であり、実態と一致していない。また、WordPress 側の `wp-content/themes/home` の存在も、リポジトリ内のテーマディレクトリとの差分を生んでいる。作業を開始する前に、`README.md` の記述を実態に合わせて修正し、前提を揃えることが調査効率向上のためにも望ましい。

## /works/ と Strength / History の論点
現在の `/works/` ページは「弊社の強み」として技術力をシネマティックに見せる構成になっているが、2026-03-25の打ち合わせ議事録では「強み」として独立させるよりも「歴史」として、会社の継承や厚みを伝える方向が優先されている。既存の `page-strength.php` や `content-works.php` の要素がこの新しい方針と乖離している可能性が高い。`/works/` のルート名を維持しつつ内容を「歴史」として再定義できるか、既存のシネマティック演出を活かせるか、また歴史ページに必要な写真や年表などの素材が不足している点が論点となっている。

## リスク
- **影響範囲の特定:** route やテンプレート名の変更が、Simply Static の静的ファイル生成、リダイレクト設定、canonical タグに予期せぬ影響を与える可能性。
- **スタイル汚染:** 共有コンポーネントやグローバルスタイルへの意図しない変更が、他のページに影響を及ぼす可能性。
- **手戻り:** 素材（写真、年表データなど）が未確定の状態で実装を進めると、後から大きな手戻りが発生するリスク。設計判断と素材待ちの分離が重要。
- **環境差異:** リポジトリと WordPress 実体 (`wp-content/themes/home`) のテーマ構成の差分が、開発中に問題を引き起こす可能性。
